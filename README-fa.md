# نظرات مستقل کوارتز (Quartz Standalone Comments)

این پروژه یک سیستم ثبت نظر را به [Quartz](https://quartz.jzhao.xyz/) اضافه می‌کند. مخزن اصلی را می‌توانید در اینجا پیدا کنید: https://github.com/dlnorman/standalone-comments.

من چندین تغییر برای بهبود سازگاری و قابلیت استفاده برای وب‌سایت‌های کوارتز ایجاد کردم.

## ویژگی‌ها

![تصویر](comments.webp)

- 🌍 پشتیبانی چندزبانه (انگلیسی و فارسی، قابل توسعه از طریق i18n)
- 😀 واکنش‌ها (Reactions) با ۱۰ گزینه ایموجی برای پست‌ها و نظرات
- 🛠️ پنل مدیریت برای مشاهده و مدیریت نظرات
- 📦 پشتیبانی از درون‌ریزی و برون‌بری (Import/Export) داده‌ها
- 🛡️ محافظت در برابر هرزنامه (Spam) و سیستم نظارت
- 📧 اعلان‌های ایمیلی برای نظرات جدید

<br>

## نصب

> ⚠️ توصیه می‌شود از PHP 8.0 یا جدیدتر استفاده کنید. اگرچه پروژه اصلی ادعا می‌کند از PHP 7.4 پشتیبانی می‌کند، من با PHP 8.1 با خطاهایی مواجه شدم. این مشکلات پس از ارتقا به PHP 8.3 برطرف شدند.

### مرحله ۱: نصب روی سرور شما

1. یک اکانت هاست اشتراکی تهیه کنید و یک ساب‌دامین بسازید، برای مثال: `comments.yourdomain.com`.
2. این ریپازتوری را با کلیک روی دکمه سبز رنگ **Code** و انتخاب **Download ZIP** دانلود کنید.
3. فایل ZIP را در پوشه `public_html` هاست خود آپلود کرده و آن را استخراج (Extract) کنید.
4. فایل `config.php` را ویرایش کنید:
   - مقدار `APP_URL` را برابر با ساب‌دامینی که فایل‌ها را در آن آپلود کرده‌اید قرار دهید، برای مثال: `https://comments.yourdomain.com`  
   - مقدار `ALLOWED_ORIGINS` را برابر با دامنه وب‌سایت کوارتز خود قرار دهید، برای مثال: `https://yourdomain.com`  
   - مقدار `APP_LANGUAGE` را برای انتخاب زبان ظاهر سایت تنظیم کنید (`en` یا `fa`)
5. فایل `set-password.php` را در مرورگر خود باز کنید:

   ```
   https://comments.yourdomain.com/set-password.php
   ```

6. یک رمز عبور برای مدیریت انتخاب کنید.
7. پنل مدیریت را باز کنید:

   ```
   https://comments.yourdomain.com/admin.html
   ```

8. رمز عبوری که انتخاب کرده بودید را وارد کنید و وارد شوید.

پس از ورود موفقیت‌آمیز، فایل `set-password.php` را از پوشه `public_html` خود حذف کنید.

### مرحله ۲: نصب افزونه کوارتز

دستور زیر را در داخل پروژه کوارتز خود اجرا کنید:

```bash
npx quartz plugin add github:fardm/quartz-standalone-comments
```

پس از نصب، فایل `quartz.config.yaml` را باز کرده و افزونه را پیکربندی کنید. مقدار `backendUrl` را برابر با آدرس سروری که سیستم نظرات را در آن آپلود کرده‌اید قرار دهید:

```yaml
- source: github:fardm/quartz-standalone-comments
  enabled: true
  options:
    backendUrl: https://comments.yourdomain.com
  layout:
    position: afterBody
    priority: 100
```

سرور پیش‌نمایش محلی را راه‌اندازی کنید:

```bash
npx quartz build --serve
```

اکنون باید بخش نظرات را در پایین صفحات خود مشاهده کنید.

به خاطر داشته باشید که این فقط یک پیش‌نمایش محلی است. از آنجا که سایت به صورت محلی اجرا می‌شود، واکنش‌ها و نظرات در پایگاه داده (دیتابیس) ذخیره نخواهند شد.

سایت خود را با استفاده از دستور زیر مستقر (Deploy) و همگام‌سازی کنید:

```bash
npx quartz sync
```

پس از استقرار، از وب‌سایت خود بازدید کرده و هم نظرات و هم واکنش‌ها را آزمایش کنید.

## ویجت آخرین نظرات

برای نمایش آخرین نظرات در سایدبار یا هر بخش دیگری از لایوت سایت، کد زیر را به فایل `quartz.config.yaml` اضافه کنید:

```yaml
- source: github:fardm/quartz-standalone-comments
  enabled: true
  options:
    backendUrl: https://comments.yourdomain.com
    type: recent
    limit: 5
  layout:
    position: right
    priority: 50
```

## فعال‌سازی اعلان‌های ایمیلی

اعلان‌های ایمیلی به یک کرون جاب (Cron Job) نیاز دارند. اگر سیستم را روی یک ساب‌دامین نصب کرده‌اید (که توصیه می‌شود)، از مسیر مطلق (Absolute Path) صحیح مربوط به ارائه‌دهنده هاست خود استفاده کنید.

1. یک کرون جاب جدید در پنل هاست خود ایجاد کنید.
2. دستور زیر را جای‌گذاری (Paste) کنید:
```
/usr/local/bin/php /home/username/domains/comments.example.com/public_html/utils/process-email-queue.php
```
3. به بخش **Utilities** بروید و **Email Notifications** (اعلان‌های ایمیلی) را فعال کنید.
4. آدرس ایمیل خود را در فیلد **Admin Email** وارد کرده و تنظیمات را ذخیره کنید.
5. چند نظر آزمایشی ایجاد کنید تا بررسی کنید همه چیز به درستی کار می‌کند.

## تغییرات من

خلاصه‌ای از تغییراتی که من در پروژه اصلی ایجاد کردم:

- افزودن پشتیبانی از پیکربندی زبان برای فرانت‌اند (ساختار آماده i18n).
- پشتیبانی از تاریخ جلالی (شمسی).
- افزودن کنترل‌های مرتب‌سازی نظرات در پنل مدیریت.
- بازنویسی سیستم درون‌ریزی/برون‌بری (Import/Export) برای شامل شدن تمامی انواع داده‌ها (نظرات، واکنش‌ها، اشتراک‌ها و هرزنامه‌ها).
- افزودن ویژگی پاکسازی پایگاه داده با یک کلیک با قابلیت انتخاب دسته‌های داده.
- بهبود استایل‌ها و رابط کاربری (UI).
- حالت تاریک (Dark mode) برای پنل مدیریت.
- طراحی مجدد واکنش‌های ایموجی (نسخه اصلی فقط شامل چهار واکنش به سبک Disqus بود؛ این نسخه انتخاب گسترده‌تری از ایموجی‌ها به سبک GitHub ارائه می‌دهد).
- نمایش راهنمای قالب‌بندی در داخل ویرایشگر نظر.

<br>

## امنیت

برای کاهش هرزنامه و سوءاستفاده، این سیستم شامل چندین لایه محافظتی است:

- محافظت هانی‌پات (Honeypot) برای شناسایی و مسدود کردن ربات‌ها
- محدودسازی نرخ درخواست (Rate limiting)
- مسدودسازی خودکار IP پس از چندین تلاش ناموفق برای ورود به پنل مدیریت
